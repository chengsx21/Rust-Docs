    通用错误¶

    一些错误是所有 API 都可能会返回的。 具体说来，每个 API 都可能返回如下错误之一：

    404 Not Found

    {
    "code": 1000,
    "info": "NOT_FOUND",
    "data": {}
}

    401 Unauthorized

    {
    "code": 1001,
    "info": "UNAUTHORIZED",
    "data": {}
}

    服务器内部错误。

    500 Internal Server Error

    {
    "code": 1003,
    "info": "INTERNAL_ERROR",
    "data": {}
}

    解析 JSON 请求体时缺少必要参数或出现错误。

    400 Bad Request

    {
    "code": 1005,
    "info": "INVALID_FORMAT",
    "data": {}
}

    USER 管理相关 /user¶

    POST /register¶

    注册一个用户。

    请求正文样例：

    {
    "user_name": "Alice",
    "password": "Hashed_Word",
    "salt": "secret_salt", 
    "mail": "mymail@163.com"
}

    正文的 JSON 包含一个字典，字典的各字段含义如下：

    字段 类型 必选 含义
    user_name 字符串 是 用户名
    password 字符串 是 用户密码
    salt 字符串 是 用户 salt
    mail 字符串 是 用户邮箱

    前端将用户名、加密后的密码与盐值传输到后端。

    后端接受请求后，首先验证用户名是否重复、合法。

    若用户名与密码都合法，则向所给邮箱发送一封邮件，并创建一个待验证用户对象等待邮件校验；否则返回错误信息。

    • 成功发送校验邮件

      成功发送用户注册校验邮件，则返回如下响应：

      200 OK

      {
    "code": 0,
    "info": "SUCCESS",
    "data": {}
}

    • 用户名重复或用户名待验证

      400 Bad Request

      {
    "code": 1,
    "info": "USER_NAME_CONFLICT",
    "data": {}
}

    • 用户名格式非法

      400 Bad Request

      {
    "code": 2,
    "info": "INVALID_USER_NAME_FORMAT",
    "data": {}
}

    • 密码格式不合法

      400 Bad Request

      {
    "code": 3,
    "info": "INVALID_PASSWORD_FORMAT",
    "data": {}
}

    POST /salt¶

    返回一个用户注册时的 salt。

    请求正文样例：

    {
    "user_name": "Alice"
}

    各字段含义如下：

    字段 类型 必选 含义
    user_name 字符串 是 用户名

    后端接受请求后，首先验证用户名是否合法，合法则返回用户 salt。

    • 成功返回 salt

      成功返回 salt，则返回如下响应：

      200 OK

      {
    "code": 0,
    "info": "SUCCESS",
    "data": {
        "salt": "SECRET_SALT"
    }
}

      各字段含义如下：

      字段 类型 必选 含义
      salt 字符串 是 用户 salt

    • 用户名或密码错误

      400 Bad Request

      {
    "code": 4,
    "info": "USER_NAME_NOT_EXISTS_OR_WRONG_PASSWORD",
    "data": {}
}

    GET /verify/[token]¶

    校验一个注册的用户。

    无需请求正文。

    后端检验 token 对应的待验证用户是否存在、未验证且处在待验证时间内。

    满足这三个条件，则注册用户、将密码进行哈希后存储并返回登录 Token，否则返回错误信息。

    • 成功校验用户

      成功校验用户，返回如下响应并转入已登录状态：

      200 OK

      {
    "code": 0,
    "info": "SUCCESS",
    "data": {
        "id": 1,
        "user_name": "Alice",
        "token": "SECRET_TOKEN"
    }
}

    • 验证用户名不存在

      400 Bad Request

      {
    "code": 15,
    "info": "INVALID_TOKEN",
    "data": {}
}

    • 用户已经验证

      400 Bad Request

      {
    "code": 16,
    "info": "ALREADY_VERIFIED",
    "data": {}
}

    • 长时间未验证需重新注册

      400 Bad Request

      {
    "code": 17,
    "info": "TOO_LONG_TIME",
    "data": {}
}

    用户登录。

    请求正文样例：

    {
    "user_name": "Alice",
    "password": "Bob19937"

    字段 类型 必选 含义
    user_name 字符串 是 用户名

    后端接受请求后，验证用户是否存在，若是则验证密码与存储的密码哈希值是否匹配。

    二者都通过则成功登录并返回 token，否则返回用户名或密码错误。

    • 成功登录

      返回如下响应并转入已登录状态：

      200 OK

      {
    "code": 0,
    "info": "SUCCESS",
    "data": {
        "id": 1,
        "user_name": "Alice",
        "token": "SECRET_TOKEN"
    }
}

    • 用户名格式非法

      400 Bad Request

      {
    "code": 2,
    "info": "INVALID_USER_NAME_FORMAT",
    "data": {}
}

    • 密码格式非法

      400 Bad Request

      {
    "code": 3,
    "info": "INVALID_PASSWORD_FORMAT",
    "data": {}
}

    • 用户名或密码错误

    POST /modifypassword¶

    用户修改密码。

    请求需要在请求头中携带 Authorization 字段，记录 token 值。

    请求正文样例：

    {
    "user_name": "Alice",
    "old_password": "Bob19937",
    "new_password": "Carol48271"
}

    正文的 JSON 包含一个字典，字典的各字段含义如下：

    字段 类型 必选 含义
    user_name 字符串 是 用户名
    old_password 字符串 是 用户旧密码
    new_password 字符串 是 用户新密码

    后端接受到请求之后，先检验 token 是否合理。

    如果合理，则判断 old_password 是否是该用户的原本密码。

    如果原密码正确，则检验新密码是否符合格式，最后进行修改。

    • 修改成功

      成功修改密码，返回如下响应：

      200 OK

      {
    "code": 0,
    "info": "SUCCESS",
    "data": {}
}

    • 用户名或密码错误

      400 Bad Request

      {
    "code": 4,
    "info": "USER_NAME_NOT_EXISTS_OR_WRONG_PASSWORD",
    "data": {}
}

    POST /comment/like/[comment_id]¶

    点赞评论。

    请求需要在请求头中携带 Authorization 字段，记录 token 值。

    请求正文无需附带内容。

    后端接受到请求之后，先检验 token 是否有效，若评论存在，且用户未对该评论点赞，则进行点赞。

    • 成功点赞

      成功点赞评论，返回如下响应：

      200 OK

      {
    "code": 0,
    "info": "SUCCESS",
    "data": {}
}

    • 评论已被点赞过

      400 Bad Request

      {
    "code": 5,
    "info": "INVALID_LIKES",
    "data": {}
}

    • 评论不存在

      400 Bad Request

      {
    "code": 11,
    "info": "COMMENTS_NOT_FOUND",
    "data": {}
}

    POST /comment/cancellike/[comment_id]¶

    取消点赞评论。

    请求正文无需附带内容。

      200 OK
